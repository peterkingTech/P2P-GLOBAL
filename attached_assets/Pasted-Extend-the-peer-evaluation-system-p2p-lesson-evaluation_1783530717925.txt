Extend the peer-evaluation system (p2p_lesson_evaluations) so evaluating others contributes real credit to the evaluator's own growth metrics, closer to how 42 School's peer-correction system works — the same reciprocity idea, adapted:

1. When an evaluator approves or requests revision on a submission (an evaluation is RESOLVED, not just assigned), award them servant_score points (from The Race system) — this is an act of service, so it belongs there, not wisdom_points (which should stay tied to the evaluator's OWN learning, not their service to others).
2. Add a new stat to p2p_profiles: evaluations_completed (integer, default 0), incremented every time this user resolves an evaluation as the evaluator. This becomes visible on their profile alongside existing stats, and is real data the Hall of Faith testimony system (p2p_testimonies) can eventually reference — don't fabricate testimony content from it, just make the raw number available.
3. Notify the assigned evaluator that they have a pending evaluation — use the existing p2p_notifications table, triggered the moment a p2p_lesson_evaluations row is created with them as evaluator_id. Without this, an evaluator has no way to know they've been assigned unless they happen to check.
4. Add a simple "Evaluations waiting for you" list/badge somewhere visible (e.g. a small counter on the Mentor tab or Home), showing how many pending evaluations are assigned to the current user.
5. Reliability safety net: if an evaluation sits in 'pending' status for more than 72 hours with no action, automatically reassign it to a different eligible evaluator (same selection logic as the original assignment — someone who's completed that lesson, least-recently-assigned) and notify the new evaluator. Log the reassignment so the original evaluator isn't penalized, just quietly relieved of it.
6. Do NOT build a point-cost/debt mechanic like 42's "lose a point when you're evaluated" — that's not needed here since our evaluators are assigned by lesson-completion eligibility, not by scarcity. The credit should be purely additive (evaluating earns you something), never a cost for being evaluated yourself.
